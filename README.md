# ANRF Flood Segmentation Phase 2 – v3 Optimized

## 1. Overview

This repository contains the Phase 2 solution for the ANRF Flood Prediction Challenge – West Bengal 2024. [file:19]  
The goal is to produce high‑resolution flood masks for each test tile, classifying pixels as non‑flooded land, flooded area, or permanent water using multi‑sensor satellite imagery. [file:19]

The main components are: [file:19][file:20]  
- `notebook3a-1.ipynb`: end‑to‑end training and inference pipeline.  
- `submission_v3.csv`: final competition submission generated by the notebook.

## 2. Notebook: `notebook3a-1.ipynb`

### 2.1 What this notebook does

The notebook implements a complete flood segmentation workflow: [file:19]

- Loads train/val/test splits and multi‑band SAR/optical GeoTIFFs.  
- Builds a 9‑channel input (6 raw bands + NDWI, MNDWI, HH/HV ratio).  
- Trains three segmentation models (UNet++, DeepLabV3+, UNet‑MiT‑B3).  
- Tunes flood probability thresholds on the validation set.  
- Runs test‑time augmentation (TTA) and 3‑model ensemble at probability level.  
- Applies morphology‑based post‑processing to clean flood masks.  
- Encodes predictions as run‑length encoding (RLE) and writes `submission_v3.csv`.

### 2.2 Key improvements vs earlier versions

This “v3 OPTIMIZED” notebook includes several critical upgrades: [file:19]

- **Test preprocessing fix**: replaces `cv2.imread()/255` with `rasterio` + the same z‑score normalization used during training, which is correct for multi‑channel SAR GeoTIFFs and removes a major train–test mismatch.  
- **9‑channel feature stack**: adds NDWI, MNDWI, and HH/HV SAR ratio as extra channels to better distinguish open water, built‑up areas, and flooded vegetation.  
- **3‑model ensemble**: combines UNet++ (EfficientNet‑B5 + scSE), DeepLabV3+ (ResNet‑50), and UNet with MiT‑B3 (SegFormer) encoder to capture both local detail and global flood context.  
- **Flood‑threshold tuning**: scans multiple flood probability thresholds on validation data to maximize flood IoU instead of using plain argmax.  
- **Morphological post‑processing**: removes tiny, isolated “flood speckles” and fills small holes inside flood regions.  
- **Pseudo‑labeling (optional Round 2)**: generates high‑confidence pseudo‑labels from test data (only where all three models agree and are very confident) and uses them for a second training round.

### 2.3 High‑level pipeline structure

Main sections in the notebook: [file:19]

1. **Setup & Config**  
   - Imports, random seed, paths (`BASEPATH`, `IMGDIR`, `LBLDIR`, `TESTDIR`, `SAVEDIR`), training hyperparameters, class weights, and inference settings.

2. **Data loading & statistics**  
   - Read train/val/test image paths and label paths.  
   - Compute per‑channel mean/std for normalization, plus extended stats when spectral indices are used.

3. **Feature engineering & dataset**  
   - `add_spectral_indices`: computes NDWI, MNDWI, HH/HV ratio.  
   - `normalize_image`: percentile clipping + z‑score per channel.  
   - `FloodDataset`: applies feature logic, albumentations, and returns tensors.

4. **Augmentation & DataLoaders**  
   - Strong train augmentations (flips, rotations, affine, elastic, noise, blur, coarse dropout).  
   - Simple validation transform.  
   - `DataLoader`s with pinned memory and persistent workers.

5. **Models, loss, metrics**  
   - `build_model1/2/3`: UNet++, DeepLabV3+, UNet‑MiT‑B3.  
   - `CombinedLoss`: class‑weighted Dice + Focal loss.  
   - `SegMetrics`: computes per‑class IoU, mIoU, and accuracy.

6. **Training & validation loops**  
   - OneCycleLR, gradient accumulation, mixed precision (`torch.amp.autocast`), gradient clipping, early stopping.  
   - Best model checkpoints saved as `bestmodel1.pth`, `bestmodel2.pth`, `bestmodel3.pth`.

7. **Flood threshold tuning**  
   - Loads best model, collects softmax probabilities on validation set.  
   - Scans flood thresholds (e.g., 0.30–0.65) and picks the best by flood IoU.  
   - Sets `CFG.FLOODTHRESHOLD` accordingly.

8. **Inference, TTA, ensemble, post‑processing**  
   - Rebuilds test list from `prediction/image` folder.  
   - For each image:  
     - Load with `rasterio`, resize, apply spectral indices, normalize.  
     - Run 3‑model TTA ensemble, average probabilities with weights (`ENSEMBLEWEIGHTS`, e.g., 0.40/0.30/0.30).  
     - Override argmax with tuned flood threshold vs water probability.  
     - Apply `postprocess_flood_mask` (connected‑component filtering + binary closing).  
     - Encode to RLE and append to submission list.

9. **Submission creation & checks**  
   - Build `DataFrame` with columns `id`, `rle_mask`.  
   - Validate:
     - `len(df) == len(test_imgs)`  
     - Unique IDs, no nulls, no empty strings.  
     - Count non‑empty masks and log flood vs no‑flood ratio.  
   - Save to `submission_v3.csv`.  
   - Randomly decode a few RLE masks and re‑encode to ensure round‑trip correctness.

## 3. Submission File: `submission_v3.csv`

### 3.1 Format

`submission_v3.csv` is the final competition submission file generated by the notebook. [file:19][file:20]  
It has one row per test tile with the following columns:

- `id`: test tile identifier (e.g., `20240529_EO4_RES2_fl_pid_082`).  
- `rle_mask`: run‑length encoded flood mask in column‑major order (Kaggle convention), where `0 0` denotes an empty mask.

### 3.2 How it was generated

For each test image: [file:19][file:20]

1. Read the multi‑band GeoTIFF using `rasterio`.  
2. Resize to 512×512 and optionally add spectral indices (NDWI, MNDWI, HH/HV).  
3. Normalize with the same channel statistics used in training.  
4. Run TTA + ensemble of the three trained models to obtain class probabilities.  
5. Apply tuned flood probability threshold and water‑class check:
   - Base prediction by argmax.  
   - Override to flood where flood prob ≥ threshold and > water prob.
6. Run morphological post‑processing to clean flood masks.  
7. Convert the binary flood mask to column‑major RLE and write to `submission_v3.csv`.

The notebook includes checks for: [file:19][file:20]

- ID coverage and uniqueness.  
- No null or empty RLE strings.  
- A reasonable number of non‑empty flood masks.  
- Encode–decode round‑trip for random samples.

## 4. How to Run

1. Upload the notebook and dataset to Kaggle (or a compatible environment). [file:19]  
2. Ensure the directory structure matches the paths in the `CFG` config (`split/*.txt`, `image/`, `label/`, `prediction/image/`).  
3. Run cells in order:
   - Install dependencies.  
   - Run imports + config.  
   - Run data, feature engineering, dataset, and loaders.  
   - Train (or load existing `bestmodel*.pth`).  
   - Run threshold tuning.  
   - Run inference section to produce `submission_v3.csv`.  
4. Download `submission_v3.csv` and submit to the ANRF / Kaggle competition. [file:19][file:20]
